For PGi, the following values are typically modified when a user is moved cross region:</para><list class="bullet"><listItem><para>	TollNumber</para></listItem><listItem><para>	TollFreeNumber</para></listItem><listItem><para>	Domain</para></listItem></list></introduction><section><title>Configure your PGi provider for regionally hosted meetings</title><content><para>To configure your PGi provider for regionally hosted meetings:</para><list class="ordered"><listItem><para>Connect to Office 365 by using Azure AD PowerShell as "Administrator" and enter the following:</para><code>Set-ExecutionPolicy RemoteSigned -Force
$cred = Get-Credential
$session = New-CsOnlineSession -Credential $cred
Import-PSSession $session –AllowClobber
</code></listItem><listItem><para></para><para>To modify the ACP settings, you must first remove the existing settings as follows: </para><code>Remove-CsUserAcp –Identity &lt;SipAddress&gt;</code><alert class="note">
 <para>If you do not first remove the existing settings, setting new values will create duplicate ACPInfo sections for the user.</para>
</alert><para></para><para>After you remove the settings, specify  the new values by using the Set-CsUserAcp cmdlet as follows.  The new settings are based on the CSV file provided by your vendor:</para><code>Set-CsUserAcp -Identity &lt;SipAddress&gt; -TollNumber "newTollNumber" –TollFreeNumber “newTollFreeNumber” -ParticipantPassCode “newPassCode”
 -Domain "newDomain" -Name "newName" -URL “newURL”</code><para></para><para>For example, the following command populates AcpInfo on a user account configured for PGi dial-in conferencing in Europe:</para><code>Set-CsUserAcp -Identity SfBUser@contoso.com -TollNumber "+4508265222" –TollFreeNumber “80251388” -ParticipantPassCode “01234567” 
-Domain "emealync.pgi.com" -Name " PGI" -URL “http://www.pgi.com/lynconline/localnums.php”</code><alert class="note">
 <para>In some cases, the ACP vendor will only supply a subset of AcpInfo XML settings and you must provide the rest. For more information, see <link xlink:href="b7064478-fc8d-4a4b-a9e5-726b638938fd#BKMK_AcpInfoSettings">AcpInfo XML settings</link>.</para>
</alert></listItem><listItem><para></para><para>Move the user to the target region. </para><alert class="note">
 <para>On premises users do not have an AcpInfo XML multi-valued attribute, so moving users from on-premises to online and modifying AcpInfo settings is not applicable.</para>
</alert><para>For more information about moving users cross region, and validating ACP functionality in the target region, see &lt;insert link: Steps for all ACPs&gt;.</para></listItem></list></content></section><section address="BKMK_AcpInfoSettings">
